NumPy: Copy vs. View

Understanding the Difference Between a New Array and a New Look

The Main Difference

Copy

A copy is a completely new array. It has its own data, and any changes made to the copy will not affect the original array.

View

A view is just a different look at the same data as the original array. Any changes made to the view will affect the original array.

The `.copy()` Method

Using .copy() creates a deep copy, a completely separate array with its own data.

Code

import numpy as np

original_arr = np.array([1, 2, 3])
# Create a completely separate copy
copied_arr = original_arr.copy()

# Modify the copy
copied_arr[0] = 99

print(f"Copied array:  {copied_arr}")
print(f"Original array: {original_arr}")

# Output:
# Copied array:  [99  2  3]
# Original array: [1 2 3]
# The original is NOT changed.

The `.view()` Method & Slicing

Using .view() or slicing an array creates a new array object that looks at the same data.

Code

import numpy as np

original_arr = np.array([1, 2, 3])
# Create a view that looks at the same data
view_arr = original_arr.view()

# Slicing also creates a view!
# sliced_view = original_arr[0:2]

# Modify the view
view_arr[0] = 99

print(f"View array:    {view_arr}")
print(f"Original array: {original_arr}")

# Output:
# View array:    [99  2  3]
# Original array: [99  2  3]
# The original IS changed.

Use copy when you need a separate, independent array. Be aware that slicing creates a view to save memory.
